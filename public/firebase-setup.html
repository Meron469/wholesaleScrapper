Firebase Firestore Setup Instructions

Your Zillow FSBO Scraper requires a Firestore database to store the scraped listings. Follow these steps to set up your database.

Note: You only need to do this setup once. After your database is created, it will be available for all future scrapes.

Step 1: Go to the Firebase Console

Visit the Firebase Firestore page for your project:

Sign in with the Google account associated with your Firebase project if prompted.

Step 2: Create your Firestore Database

Click the "Create database" button to begin setting up your Firestore database.

Step 3: Select Security Rules

You'll be presented with two options:

  • Start in production mode - More secure, but requires rules setup
  • Start in test mode - Allows read/write access for easier testing

For now, select "Start in test mode" for easier setup. You can change this later if needed.

Step 4: Select a Database Location

Choose a location where your data will be physically stored. For best performance, select a location close to where most of your users are located.

The default recommendation is usually fine for most use cases.

Step 5: Enable the Database

Click the "Enable" button to create your Firestore database.

This process may take a minute or two to complete.

Step 6: Wait for Changes to Propagate

After creating your database, it may take a few minutes for the changes to fully propagate through Google's systems.

Please wait 2-3 minutes before testing your application.

Step 7: Verify Your Setup

After waiting, refresh this page to check if your Firestore database is properly connected.

Troubleshooting: If you still see errors after completing these steps, try:
  • Waiting a bit longer (sometimes propagation can take 5-10 minutes)
  • Double-checking that the API is enabled at Google Cloud Console
  • Verifying that your service account has the correct permissions in IAM settings
Next Steps: Once your Firestore database is set up, you'll be able to store and retrieve scraped Zillow listings automatically.
